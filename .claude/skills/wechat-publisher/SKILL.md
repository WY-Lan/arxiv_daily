# 微信公众号发布技能

将AI Agent论文转化为适合微信公众号的深度技术文章，支持单篇详解和合集汇总。

## 发布模式

### 模式一：单篇发布 (single)

适用于重点论文的深度解读文章。

**输出格式**：
```json
{
  "title": "标题（15-25字，专业且吸引）",
  "summary": "摘要（100字以内，用于封面展示）",
  "content": "正文（Markdown格式，2000-3000字）",
  "images": [
    {
      "position": "开头/方法/实验/结尾",
      "description": "配图建议",
      "content": "图片内容描述"
    }
  ],
  "toc": ["目录项1", "目录项2", "目录项3"],
  "reading_time": "预计阅读时间"
}
```

### 模式二：合集发布 (collection)

适用于每日多篇论文的汇总推送。

**输出格式**：
```json
{
  "title": "标题（如：今日AI Agent论文精选 | 5篇）",
  "summary": "摘要",
  "content": "正文（含多篇论文简介）",
  "images": [],
  "toc": [],
  "reading_time": "预计阅读时间"
}
```

## 文章结构模板

### 深度解读结构（单篇）

```
1. 引言（10%）
   - 背景介绍
   - 问题引入
   - 本文价值

2. 研究背景（15%）
   - 相关工作
   - 现有问题
   - 研究动机

3. 核心方法（35%）
   - 整体框架
   - 关键技术
   - 创新点解析

4. 实验结果（25%）
   - 数据集/基准
   - 主要结果
   - 消融实验

5. 讨论与展望（15%）
   - 局限性
   - 未来方向
   - 总结
```

### 合集结构（多篇）

```
1. 导语：今日论文概览
2. 论文列表：
   - 标题 + 标签
   - 核心贡献（3-5句话）
   - 创新亮点
   - 关键数据
   - 论文链接
3. 总结：今日洞察与展望
```

## 写作风格指南

### 专业性
- 技术术语准确
- 复杂概念需解释
- 适当使用类比

### 可读性
- 使用小标题分段
- 论点要有论据
- 结论要有总结

### 吸引力
- 开头要抓人
- 中间有亮点
- 结尾有启发

## 微信排版规范

- 标题用 `##`，副标题用 `###`
- 段落间空行
- 重要内容加粗
- 列表用 `-` 或数字
- 代码块用 ` ```语言 `
- 公式用 LaTeX 格式

## 技术集成

### API接口
- 获取 access_token：`/cgi-bin/token`
- 上传图片：`/cgi-bin/material/add_material`
- 创建草稿：`/cgi-bin/draft/add`

### 封面图片
- 推荐尺寸：900×383px（2.35:1）
- 格式：JPG/PNG
- 需要先上传获取 media_id

### 注意事项
- 需要配置 IP 白名单
- access_token 有效期 2 小时
- 草稿保存后在后台可编辑
